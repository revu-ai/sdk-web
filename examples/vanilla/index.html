Identity

The SDK auto-assigns a persistent userId on first visit. On login, call revu.identify(realId) to replace it (emits $identify); on logout, revu.reset() rotates to a fresh anonymous id (emits $reset).

Current user: (none)

When a previously-anonymous visitor signs up, revu.alias(authoritativeId) merges the current anonymous and user ids under one authoritative id (emits $alias).

Form submit

Submits emit $form_submit with field metadata only: field_names, field_types, field_count. Values are never read. Changes to a control emit $change with the control type (and checked for checkboxes/radios), never the entered value.

Right-click

Right-click the target to emit $rightclick with a fingerprint of the target element. Useful for tracking context-menu exposure and "open in new tab" intent.

Redaction with data-revu-mask

Any element inside a data-revu-mask subtree is opted out of fingerprinting: the captured event keeps the tag, role, and a fragile selector, but drops text, aria-label, and title. Click both buttons below and compare the rows in the event stream: the unmasked one's fingerprint.text reads "Visible label"; the masked one's is absent.

Outbound + download links

Links whose hostname differs from the page emit $outbound_link. Links with a download attribute (or a known file extension) emit $file_download.

bfcache restore

When the browser restores this page from the back-forward cache (you navigate away in the same tab, then press Back), the SDK emits $page_restore. This lets the server distinguish a fresh page render from a cached restoration for engagement metrics. Custom events

For signals autocapture cannot see (a server-side completion, an async export, a wizard step that does not change the URL), call revu.capture(name, properties?).

Consent (categories + GPC)

Capture is gated on a per-category consent state. Only analytics gates capture; marketing and functional are declarative and ride along on every event as $consent for the server to honor. optOut() / optIn() are aliases for denying / granting analytics. Watch the Engine state panel below update as you toggle.

Attribution (first touch + last touch)

The SDK persists the campaign that first acquired the visitor ($initial_utm_*) and the most recent one ($utm_*), so a conversion pages or days later still carries them. To see it, reload this page with campaign params, then click anything and watch the Engine state panel:

Selector filtering + beforeSend

#do-not-track is in autocaptureDenySelectors, so clicking it emits nothing (compare the unmasked button, which redacts but still emits). debug_noise is dropped by the beforeSend hook. Every event that does ship is enriched by beforeSend with properties.demo_enriched = true.
